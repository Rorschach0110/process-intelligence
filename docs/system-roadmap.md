# 成熟工业系统演进规划

## 愿景

Process Intelligence 的目标不是停留在“上传 CSV 后展示一次结果”的演示工具，而是演进为一个可持续接入工业数据、分析流程、核算碳排、沉淀知识和支撑优化决策的工业系统。

成熟形态应覆盖：

```text
数据接入 -> 数据治理 -> 流程挖掘 -> 碳排核算 -> 知识图谱 -> 优化决策 -> 仿真评估 -> 报告输出 -> 持续监控
```

每个板块都应是独立功能模块，同时共享数据集、任务、权限、历史和报告体系。

## 功能模块

### 数据中心

- CSV、Excel、JSON 上传。
- MES、ERP、SCADA、能耗系统接口接入。
- 字段映射模板、数据质量检查、数据版本和数据血缘。
- 记录上传人、来源系统、映射规则、校验结果和关联分析任务。

### 流程挖掘中心

- 案例路径、活动频次、直接跟随关系和流程变体。
- 活动执行时间、等待时间、瓶颈、返工循环和合规偏差。
- 支持班组、产线、时间段、优化前后等流程对比。

### 碳排核算中心

- 电力、物料、设备、工序、区域电网和时间段因子。
- Scope 1、Scope 2、Scope 3 分类。
- 因子版本管理。
- 活动、资源、订单、产品维度碳排和单位产量碳强度。

### 知识图谱中心

- 标准图谱 JSON schema 优先。
- Process、Case、Activity、Resource、Device、Material、Product、CarbonMetric、Bottleneck、Recommendation 建模。
- Neo4j 作为可选适配器，而不是核心强依赖。

### 优化决策中心

- 规则优化器稳定兜底。
- LLM provider 负责解释、方案生成和报告语言。
- 仿真或算法优化器负责 What-if 评估。
- 每条建议包含目标、证据、动作、减排量、产能影响、置信度、风险和验证方式。

### 仿真评估中心

- 优化前后对比。
- 多方案 What-if 场景。
- 减排、产能、成本、风险四类指标。
- 输出方案优先级和实施建议。

### 报告中心

- HTML、PDF、Word 报告。
- 管理层摘要、工程师明细、碳管理报告和设备维护报告。
- 包含图谱截图、碳排明细、优化清单、数据质量和方法学说明。

### 任务与历史中心

- 所有分析都作为任务记录。
- 记录输入数据、参数、字段映射、碳因子版本、运行状态、日志、结果文件和创建时间。
- 从 JSON 文件演进到 SQLite 或 DuckDB。

### 权限与组织中心

- 用户、角色、工厂、车间、产线和项目。
- 最小角色：Admin、Engineer、Viewer。
- 数据权限、报告权限和操作审计。

### 系统运维中心

- 健康检查、任务日志、错误追踪、备份恢复、配置管理、API 记录和诊断包导出。

## 技术路线

建议目标架构：

```text
frontend: Next.js + React + TypeScript + Tailwind CSS
backend: FastAPI + service layer + repository layer + task runner
storage: SQLite 或 DuckDB + 文件制品存储
graph: JSON schema first + optional Neo4j adapter
optimizer: rule provider + OpenAI-compatible provider + simulation provider
```

引入顺序应保持克制：

```text
FastAPI + SQLite/DuckDB -> Next.js -> 任务系统 -> Neo4j adapter -> 真实 LLM
```

## 阶段规划

### P7 - 架构升级基础

- 引入 FastAPI 服务入口。
- 引入 SQLite 或 DuckDB 作为最小持久化。
- 建立数据集、上传、运行任务和结果摘要表。
- 拆分 router、service、repository 层。
- 保留现有标准库后端作为兼容参考，直到新入口稳定。

### P8 - 数据中心

- 数据集管理。
- 字段映射模板。
- 数据质量报告。
- 上传历史。
- 数据预览分页。

### P9 - 流程挖掘中心

- 等待时间和执行时间拆分。
- 变体对比。
- 返工路径分析。
- 流程健康评分。
- 固化流程分析结果 schema。

### P10 - 碳排核算中心

- 碳因子库。
- 因子版本。
- 多维碳排聚合。
- 产品、订单、资源碳强度。
- 碳核算方法说明。

### P11 - 图谱中心

- 扩展图谱 schema。
- 图谱查询 API。
- 图谱导出。
- 可选 Neo4j adapter。
- 前端图谱交互增强。

### P12 - 优化中心

- 增强规则优化器。
- LLM provider 可配置。
- 优化建议审核。
- 建议版本管理。
- 置信度、风险和影响评估。

### P13 - 仿真与方案比较

- What-if 参数。
- 多方案对比。
- 减排、产能、成本三目标评估。
- 优先级排序。

### P14 - 报告中心

- 报告模板。
- HTML、PDF 导出。
- 管理层摘要。
- 工程明细报告。
- 审计说明。

### P15 - 工业化部署

- 用户与角色。
- 操作日志。
- 配置中心。
- Docker 部署。
- 备份恢复。
- 监控与诊断。

## 当前架构风险

- API 路由继续膨胀会带来刚性和脆弱性。
- JSON 文件历史适合原型，但不适合复杂查询、权限和审计。
- 前端仍是单页原型，业务模块边界不明显。
- 直接一次性接入 Neo4j、LLM、任务队列和 Next.js 会让系统过早复杂。

因此 P7 应优先建设后端分层和最小数据库底座，再逐步产品化每个功能板块。
