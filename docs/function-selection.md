# 功能选择说明

新平台目标是“流程挖掘 -> 多尺度碳排量化 -> 知识图谱建模 -> AI 优化”的闭环，因此从旧项目中保留的是业务能力，而不是完整工程结构。

## 保留

- 流程挖掘：事件日志、活动、变体、直接跟随关系、瓶颈。
- 多尺度碳排量化：活动级、资源级、总量级碳排聚合。
- 知识图谱建模：Process、Activity、Resource、Metric 节点和关系。
- AI 优化：以流程证据和碳排结果为依据输出优化建议。

## 暂不保留

- 登录注册、工作区管理、旧数据库表。
- BPMN 绘图器、Dotted Chart、完整统计分析页面。
- 3D 资产管理。
- 能耗知识库文档 CRUD。
- DeepSeek/Qwen 真实 API 调用。
- Neo4j、MySQL、Milvus、PM4Py 等重依赖。

## 为什么这样裁剪

第一版目标是能跑通闭环，而不是复刻旧平台。旧项目很多功能有价值，但直接迁移会带来复杂依赖、硬编码配置和环境搭建成本。MVP 先把数据链路、接口形态和前端工作台跑起来，再逐步把重功能接回来。
